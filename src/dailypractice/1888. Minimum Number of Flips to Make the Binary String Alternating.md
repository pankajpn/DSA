You are given a binary string s. You are allowed to perform two types of operations on the string in any sequence:

Type-1: Remove the character at the start of the string s and append it to the end of the string.
Type-2: Pick any character in s and flip its value, i.e., if its value is '0' it becomes '1' and vice-versa.
Return the minimum number of type-2 operations you need to perform such that s becomes alternating.

The string is called alternating if no two adjacent characters are equal.

For example, the strings "010" and "1010" are alternating, while the string "0100" is not.

Example 1:

Input: s = "111000"
Output: 2
Explanation: Use the first operation two times to make s = "100011".
Then, use the second operation on the third and sixth elements to make s = "101010".
Example 2:

Input: s = "010"
Output: 0
Explanation: The string is already alternating.
Example 3:

Input: s = "1110"
Output: 1
Explanation: Use the second operation on the second element to make s = "1010".
 

Constraints:

1 <= s.length <= 105
s[i] is either '0' or '1'.
## ✅ Sliding Window Solution (Minimum Type-2 Flips)

To minimize type-2 flips, we can use **type-1 rotations freely** to choose the best alignment of the string.

### Key observation
- Rotating the string is equivalent to selecting a **window of length `n`** from a doubled string `s + s`.
- For each such window, we need to compute how many characters differ from an alternating pattern.
- There are only two alternating patterns to consider:
  - Pattern A: `"010101..."`
  - Pattern B: `"101010..."`

### Sliding window approach (O(n))
1. Build `t = s + s` (length `2n`) so rotations correspond to contiguous windows of length `n` in `t`.
2. Maintain mismatch counts for Pattern A and Pattern B using a sliding window.
3. For each window of size `n`, the minimum mismatch count between the two patterns is the answer for that rotation.
4. The global minimum over all windows is the final answer.

### Dry run (example: `s = "111000"`)
- `s + s = "111000111000"` (window size = 6)
- Pattern A (start with 0): `010101`
- Pattern B (start with 1): `101010`

We slide a window of length 6 and track mismatches: (only key steps shown)

1) Window `"111000"`:
   - Compare to Pattern A: mismatches = 4
   - Compare to Pattern B: mismatches = 2
   → best = 2

2) Rotate once (window `"110001"`):
   - Pattern A mismatches change by removing left char and adding right char.
   - Pattern B mismatches similarly.
   → best for this window stays >= 2

After checking all rotations, minimum flips = 2 (as expected).

### Java implementation (sliding window)
```java
package dailypractice;

public class MinimumNumberOfFlipsToMakeTheBinaryStringAlternating {
    public int minFlips(String s) {
        int n = s.length();
        if (n <= 1) return 0;

        String t = s + s;
        // We'll count mismatches for the two possible alternating patterns.
        int mismatchesA = 0; // expects '0' at even indices (pattern "0101...")
        int mismatchesB = 0; // expects '1' at even indices (pattern "1010...")
        int res = n; // worst-case flips is n

        for (int i = 0; i < 2 * n; i++) {
            char c = t.charAt(i);
            // expected chars at position i for the two patterns
            char expectedA = (i % 2 == 0) ? '0' : '1';
            char expectedB = (i % 2 == 0) ? '1' : '0';

            if (c != expectedA) mismatchesA++;
            if (c != expectedB) mismatchesB++;

            // Once we have a full window of size n, start sliding
            if (i >= n) {
                // remove the effect of the outgoing char
                char out = t.charAt(i - n);
                char outExpectedA = ((i - n) % 2 == 0) ? '0' : '1';
                char outExpectedB = ((i - n) % 2 == 0) ? '1' : '0';
                if (out != outExpectedA) mismatchesA--;
                if (out != outExpectedB) mismatchesB--;
            }

            // When the window size reaches n, consider updating result
            if (i >= n - 1) {
                res = Math.min(res, Math.min(mismatchesA, mismatchesB));
            }
        }

        return res;
    }
}
```

> **Note:** This solution runs in **O(n)** time and **O(n)** space (for the doubled string). The sliding window ensures we efficiently reuse counts instead of recomputing mismatches from scratch for each rotation.
